# v0.1.0 Release Notes

## Positioning

This release is a Windows-first sequential v1 baseline for local orchestration and scaffold work. It is designed to be honest about current behavior rather than overclaim future product scope.

## Included now

- reuse-first local CLI discovery via `doctor`
- private `/solo` packet generation
- sequential `/orch` execution with per-lane reviews and consolidated verdicts
- lane presets with `default-planners`
- explicit optional write-capable `opencode-minimax-build-thinking`
- operator runbook and provider compatibility matrix
- CI and enforcement gates aligned with the current repo truth

## Explicitly not in this release

- debate loop or parallel fan-out
- dedicated OS-window spawning per lane
- native slash-mode patching inside Codex itself

## Verification summary

- local `make check`
- installed console-script smoke
- Windows primary CI lane
- bounded live smokes for supported providers and default planner lanes on this machine
